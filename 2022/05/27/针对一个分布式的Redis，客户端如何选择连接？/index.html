针对一个分布式的Redis，客户端如何选择连接？

2022-05-27 Redis Proxy, Redis 评论

客户端的几种模式

当我们按照AKF原则构建了分布式的Redis，存在着众多的节点，那么此时客户端怎么设计？

数据可以分类，交集不多

数据没办法来拆分

hash + 取模

弊端

取模的数必须固定，不能更改
影响分布式下的扩展性

random

一致性hash

客户端的连接方式

直连

针对每个客户端都需要连接所有的Redis节点，这种方式对Redis Server来说还是有很大成本，连接成本高。

代理

在客户端和Redis Server中间添加一层代理，由代理连接所有的Redis节点，而Redis客户端直接连接代理就可以。对于代理层要求比较高，性能高且能高可用。

代理的转发策略

代理层可以使用modula、random和一致性hash来选择Redis节点。
